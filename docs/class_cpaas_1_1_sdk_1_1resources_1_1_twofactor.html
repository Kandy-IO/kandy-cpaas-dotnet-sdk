TwofactorResponse Cpaas.Sdk.resources.Twofactor.ResendCode ( dynamic  destinationAddress,
Dictionary< string, string >  values 
)
inline

Resending the authentication code via same code resource, invalidating the previously sent code.

Parameters
destinationAddressstring | Array<string> Destination address of the authentication code being sent. For sms type authentication codes, it should contain a E164 phone number. For e-mail type authentication codes, it should contain a valid e-mail address.
valuesDictionary<string, string> A dictionary to hold all parameters.
  • values["codeId"] string ID of the authentication code.
  • values["message"] string Message text sent to the destination, containing the placeholder for the code within the text. CPaaS requires to have *{code}* string within the text in order to generate a code and inject into the text. For email type code, one usage is to have the *{code}* string located within the link in order to get a unique link.
  • values["subject"] string Sender address information, basically the from address. E164 formatted DID number passed as a value, which is owned by the user. If the user wants to let CPaaS uses the default assigned DID number, this field can either has "default" value or the same value as the userId.
  • values["method"] string Optional Default - sms. Type of the authentication code delivery method, sms and email are supported types. Possible values: sms, email
  • values["expiry"] string Optional Default - 120. Lifetime duration of the code sent in seconds. This can contain values between 30 and 3600 seconds.
  • values["length"] string Optional Default - 6. Length of the authentication code tha CPaaS should generate for this request. It can contain values between 4 and 10.
  • values["type"] string Optional Default - "numeric". Type of the code that is generated. If not provided, default value is numeric. Possible values: numeric, alphanumeric, alphabetic
Returns
TwofactorResponse
